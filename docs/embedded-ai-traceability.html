  • Radial chart layouts can be intuitive for “getting-to-grips” with what’s going on in an agent session.
    • In our example it took several sessions to get the Forge App from earlier, BUT
    • For illustration purposes, I’ve added a single Rovo Dev session log here…
  • If you’ll let me explain
  • D3’s radial tree, is a circular tree-layout plot…
    • the central point provides the user / agent perspective
    • the start of the chart (it’s first messages) appear just to the right of 12 o’clock
    • subsequent messages continue round in a clockwise manner
  • Edges are coloured BLUE for a request (from me, or a tool), RED for a response
    • Nodes are coloured coded by type (system / user-prompt, text, tool-call etc.)
      • calls to the LLM (in this case Anthropic) are FURTHER identified with a GREY circle surrounding the node
  • We can explore what’s happened, by hovering over the edges and nodes of the chart
